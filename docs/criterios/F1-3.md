```gherkin
Funcionalidade: Cadastro de aluno — lógica
Regra de negócio: cadastrar um aluno cria um Usuario (tipo ALUNO) e um Aluno vinculado, com senha em hash e código único gerado automaticamente.

  Cenário: cadastro com dados válidos cria Usuario e Aluno
    Dado nenhum usuario cadastrado
    Quando um POST é enviado para cadastro_aluno com nome, email, senha, genero, serie, escola e tipo_escola válidos
    Então um Usuario com tipo ALUNO é criado no banco com a senha em hash
    E um Aluno vinculado a esse Usuario é criado com codigo no formato E + 6 chars

  Cenário: cadastro redireciona após sucesso
    Dado nenhum usuario cadastrado
    Quando um POST válido é enviado para cadastro_aluno
    Então a resposta é um redirect (status 302)

  Cenário: email duplicado não cria nada
    Dado um Usuario já cadastrado com email "ana@exemplo.com"
    Quando um POST é enviado para cadastro_aluno com o mesmo email
    Então nenhum novo Usuario é criado
    E a resposta tem status 200 com o formulário e erro visível

  Cenário: campo obrigatório ausente não cria nada
    Dado nenhum usuario cadastrado
    Quando um POST é enviado para cadastro_aluno sem o campo email
    Então nenhum Usuario é criado
    E a resposta tem status 200 com o formulário e erro visível
```
