Funcionalidade: Cadastro de administrador - lógica
Regra de negócio: cadastrar um administrador cria um Usuario (tipo ADM) e um Administrador vinculado não aprovado, que não consegue logar

  Cenário: cadastro com dados válidos cria Usuario e Administrador
    Dado nenhum usuario cadastrado
    Quando um POST é enviado para cadastro_administrador com email, nome e senha válidos
    Então um Usuario com tipo ADM é criado no banco com a senha em hash
    E um Administrador vinculado a esse Usuario é criado com aprovado=False

  Cenário: cadastro redireciona após sucesso
    Dado nenhum usuario cadastrado
    Quando um POST válido é enviado para cadastro_administrador
    Então a resposta é um redirect (status 302)

  Cenário: email duplicado não cria nada
    Dado um Usuario já cadastrado com email "ana@exemplo.com"
    Quando um POST é enviado para cadastro_administrador com o mesmo email
    Então nenhum novo Usuario é criado
    E a resposta tem status 200 com o formulário e erro visível

  Cenário: campo obrigatório ausente não cria nada
    Dado nenhum usuario cadastrado
    Quando um POST é enviado para cadastro_administrador sem o campo email
    Então nenhum Usuario é criado
    E a resposta tem status 200 com o formulário e erro visível
