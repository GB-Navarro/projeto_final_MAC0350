Funcionalidade: Cadastro de aluno — página
Regra de negócio: a página de cadastro exibe todos os campos obrigatórios e opcionais, e mostra erros de validação visíveis ao usuário.

  Cenário: GET exibe o formulário com todos os campos
    Dado nenhum usuario autenticado
    Quando um GET é feito para cadastro_aluno
    Então a resposta tem status 200
    E o HTML contém campos para nome, email, senha, genero, serie, escola e tipo_escola

  Cenário: erro de validação exibe mensagem visível
    Dado nenhum usuario autenticado
    Quando um POST é enviado para cadastro_aluno sem o campo email
    Então a resposta tem status 200
    E o HTML contém uma mensagem de erro visível
